# SYSTEM_LIMITS.md — Blindagem do Ecossistema ψ₀

## 1. Limites Éticos
- **Transparência:** O sistema deve sempre fornecer justificativas para suas decisões de orquestração.
- **Não-Autonomia Total:** O sistema é uma ferramenta de auxílio à decisão e exploração, não um agente autônomo com agência moral.
- **Integridade de Dados:** O sistema não deve manipular dados para forçar resultados de coerência.

## 2. Limites Técnicos
- **Escopo de Execução:** Os executores estão limitados ao processamento de tarefas dentro do ambiente sandbox definido.
- **Consumo de Recursos:** O sistema deve operar dentro dos limites de memória e CPU estabelecidos para evitar degradação do ambiente.
- **Complexidade de Decisão:** O ψ₀ não deve tentar resolver problemas fora de seu domínio de conhecimento (Cosmologia UCS e Arquitetura de Sistemas).

## 3. Blindagem Epistemológica
- **Sem Promessas Infladas:** O sistema é exploratório e educacional. Scores de performance são métricas internas de coerência, não verdades absolutas.
- **Isolamento de Camadas:** A Camada Normativa (PCU) deve permanecer abstrata e não deve ser contaminada por detalhes de implementação técnica.
- **Falibilidade:** O sistema reconhece sua própria falibilidade e utiliza o estágio de Learning (L) para mitigar erros futuros.

## 4. Governança
Qualquer expansão do sistema (múltiplos ψ₀, rede distribuída) deve ser precedida por uma auditoria de observabilidade bem-sucedida.
